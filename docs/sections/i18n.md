The panel's interface text is translatable. Strings run through vue-i18n on the frontend, sourced from Laravel language files, so you localize the panel the same way you localize the rest of your app.

### How it works

The panel follows your application's active locale (`app()->getLocale()`). On each request it shares the resolved translation strings with the frontend, where vue-i18n renders them. Because the messages come from server-side lang files, you can add or override translations without rebuilding the panel's assets.

### Overriding and adding locales

Publish the language files:

```bash
php artisan vendor:publish --tag=saddle-lang
```

This copies `panel.php` into `lang/vendor/saddle/en/`. Edit it to change the English strings, or add a locale by creating `lang/vendor/saddle/{locale}/panel.php` (for example `lang/vendor/saddle/fr/panel.php`). Set your app's locale and the panel follows.

### What ships

Saddle ships English. The keys cover the panel's own chrome (buttons, row controls, empty states, confirmations, notifications, and flash messages). Your resource labels, field labels, and data are yours to translate through your application as usual.
